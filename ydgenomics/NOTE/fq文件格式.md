# fq 文件的特点
> 简述fq文件的特点，以及不同版本fq的格式
FASTQ（或.fq）是高通量测序中最常用的文件格式，用于同时存储DNA序列的碱基信息及其对应的测序质量得分。

### 📄 核心特点
- **文本格式与四行结构**：文件为纯文本，每个测序读长（read）的信息由**4行**组成：
    1.  **标识行**：以 `@` 开头，包含测序仪ID、位置等信息。
    2.  **序列行**：具体的碱基序列（A, T, C, G, N）。
    3.  **分隔行**：以 `+` 开头，后可跟描述信息（通常为空）。
    4.  **质量行**：与第二行碱基一一对应的质量值，用ASCII字符编码。
- **质量编码**：质量值用ASCII字符表示，核心公式为 `Q = -10 log10(Pe)`，数值越高代表该位置碱基测序越可靠。

### ⚙️ 不同版本格式（质量编码的区别）
FASTQ格式在发展过程中出现了几个互不兼容的版本，**核心区别在于质量分数的类型及ASCII码的偏移量**。目前绝大多数新数据（如Illumina 1.8+）都已统一采用**Sanger**标准。

下表整理了三种主要变体的关键参数：

| 版本名称 | ASCII偏移量 | 质量分数类型 | 分数范围 | ASCII范围 | 当前状态 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **Sanger** (标准) | **33** | PHRED (Q值) | 0 到 93 | 33-126 | **主流标准** (Illumina 1.8+) |
| **Illumina 1.3+** | **64** | PHRED (Q值) | 0 到 62 | 64-126 | **已淘汰** (1.3至1.7版本) |
| **Solexa** (早期) | **64** | Solexa (负值允许) | -5 到 62 | 59-126 | **已淘汰** (1.0及更早) |

**关键区别解读**：
- **Sanger vs. Illumina 1.3+**：两者都使用PHRED分数，但**偏移量不同**。例如，质量分40在Sanger格式中对应ASCII码 `73`（字符 `I`），而在Illumina 1.3+格式中对应ASCII码 `104`（字符 `h`）。
- **Solexa（早期）**：使用的是自定义的对数质量分（可负），与PHRED分数不可直接通用。

处理FASTQ文件时，如果混淆了不同版本的编码规则，会导致错误解读测序质量（如将低质量误判为高质量）。因此，**首先确认文件的质量编码版本至关重要**。

### 3. Solexa 早期格式 (偏移量 64，分数体系不同)
**关键区别**：该版本不使用 PHRED 公式，而是 Solexa 自定义的对数分数（允许负值）。  
**映射关系**（仅示例，非完整线性表）：

| 字符 | ASCII码 | Solexa 分数 | 约等 PHRED Q |
|------|---------|-------------|--------------|

| `?` | 63 | 0 | ~0 |
| `@` | 64 | 1 | ~1 |
| `A` | 65 | 2 | ~2 |
| ... | ... | ... | ... |
| `h` | 104 | 40 | ~40 |

> **实用建议**：遇到 Solexa 格式，应使用工具（如 `fastq_quality_converter`）转换为 Sanger 格式，避免直接按 PHRED 解释。

| 格式 | Q=30 对应字符 | Q=40 对应字符 |
|------|--------------|--------------|
| **Sanger** | `?` (ASCII 63) | `I` (ASCII 73) |
| **Illumina 1.3+** | `^` (ASCII 94) | `h` (ASCII 104) |
| **Solexa** | 通常对应 `^` 或附近字符 | 通常对应 `h` 或附近字符（但分数定义不同） |

> 1. 同一字符在不同版本中代表不同质量（如 `h` 在 Sanger 中 Q=71，在 Illumina 1.3+ 中 Q=40）。  
> 2. 处理 FASTQ 文件前，**务必先确认其质量编码版本**，可使用 `FastQC` 或 `fastq_validate` 等工具检测。

<details> <summary> FASTQ文件的质控与检查 </summary>

FASTQ文件的质控与检查是测序数据分析的第一步，也是最关键的一步。这里整理了从基础检查到深度过滤的完整方案，并附带了可直接运行的代码。

### 🔍 第一步：概览性检查

在进行任何过滤前，先要了解数据的基本情况。

-   **FastQC (最经典)**：生成详细的HTML可视化报告，包含碱基质量、GC含量、接头污染等核心指标。
    ```bash
    # 安装 (Linux)
    sudo apt-get install fastqc
    # 运行
    fastqc /data/work/TEST/fq/0A1R_w4q20m35_N_converted_1.fq -o fastqc/
    ```

-   **fastqe (趣味教学)**：将质量分数转化为Emoji表情输出在终端，适合快速查看或教学。
    ```bash
    # 安装
    pip install fastqe
    # 运行 (显示平均质量)
    fastqe input.fastq --mean
    ```

-   **Rfastp (统计整合)**：直接读取`fastp`生成的JSON报告，将结果转化为R语言数据框，方便后续统计分析。
    ```r
    library(Rfastp)
    # 假设你已经用 rfastp() 函数处理过文件，并得到了json报告路径
    stats <- qcSummary("fastp_report.json")
    print(stats)
    ```

### ✂️ 第二步：自动化过滤与修剪

对于常规数据清洗，建议使用**fastp**。它集成了质量过滤、接头修剪、长度筛选等功能，且速度极快，输出报告也很直观。

-   **基础用法 (单端数据)**：
    ```bash
    fastp -i raw_data.fastq -o clean_data.fastq
    ```

-   **进阶用法 (双端数据，启用多项过滤)**：
    以下命令是一套比较标准的质控流程，可以处理大多数Illumina数据。
    ```bash
    fastp -i R1.fastq.gz -I R2.fastq.gz \
          -o clean_R1.fastq.gz -O clean_R2.fastq.gz \
          --detect_adapter_for_pe \       # 双端自动检测接头
          --cut_front --cut_tail \        # 滑动窗口切除首尾低质量碱基
          --cut_window_size 4 \           # 窗口大小为4bp
          --cut_mean_quality 20 \         # 窗口平均质量低于20则切除
          --length_required 50 \          # 丢弃长度小于50bp的reads
          --average_qual 20 \             # 整条reads平均质量低于20则丢弃
          --correction \                  # 通过重叠区校正碱基
          --thread 8 \                    # 启用8线程
          --html report.html \            # 输出网页报告
          --json report.json              # 输出JSON数据
    ```

### 🎯 第三步：特定场景的定制化过滤

如果你的数据来自特定平台或实验设计，可能需要针对性处理。

-   **处理双端数据**：在第二步基础上，若希望专门处理未配对的reads，可使用`--unpaired1`和`--unpaired2`参数保存它们。
-   **处理Nanopore数据**：可使用`qcat`等工具进行按barcode拆分（demultiplexing）。
    ```bash
    qcat -f input.fastq -b output_barcodes/ --detect-middle
    ```
-   **结构验证 (如Capture-SELEX)**：如果数据有预期的固定结构（如恒定区），可以使用`selexqc`进行精确的结构验证和过滤。
    ```bash
    # 验证恒定区及上下游长度
    selexqc -i library.fq.gz -o results -c CONSTANT_SEQ --upstream-length 40 --downstream-length 10 --filter
    ```

### 📦 第四步：整合与可视化

在处理完多个样本后，推荐使用**MultiQC**将所有FastQC或fastp的报告汇总成一个综合报告，方便整体评估。

```bash
# 安装
pip install multiqc
# 运行 (会搜索当前目录下所有FastQC/fastp的zip或json报告)
multiqc ./
```

### 💎 总结

1.  **日常通用**：直接用`fastp`走一遍第二步的进阶命令，它涵盖了绝大多数质控需求，并直接输出清洗后的数据。
2.  **深度分析**：在处理前先使用`FastQC`查看原始数据质量，可以帮助你判断是否需要开启`fastp`中的特定选项。
3.  **特殊项目**：对于ONT或特定的靶向测序数据，需要配合`qcat`或`selexqc`等专业工具。
